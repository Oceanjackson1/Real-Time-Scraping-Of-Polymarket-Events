## 一、数据采集方法论与口径说明

### 1.1 数据获取流程

本报告数据来自自研的 Polymarket 爬虫系统，通过 Gamma API 分页拉取全量活跃事件数据。核心流程：

```
Gamma API → 分页请求 (100条/页) → 原始 JSON → 数据解析 → 事件去重 → 结构化产出
```

- **分页策略**: 按 `volume24hr` 降序排列，每页 100 条，最多拉取 50 页
- **去重机制**: 基于 `event.id` 去重，确保每个事件唯一
- **交易量计算**: 事件交易量 = Σ(所有子市场交易量)，而非 API 返回的事件级字段

### 1.2 预言机类型识别逻辑

系统通过以下规则判定预言机类型：

| 判定条件 | 预言机类型 | Oracle Link 构造方式 |
|---------|-----------|-------------------|
| `umaBond` 字段不为 `null` | **UMA** | `https://polygonscan.com/address/{resolvedBy}` |
| `umaBond` 为 `null`，描述中包含 `data.chain.link` URL | **Chainlink** | 从市场描述中正则提取 Chainlink 数据源链接 |

### 1.3 重要口径说明：Chainlink 交易量的时间窗口偏差

**本报告中 Chainlink 相关事件的交易量数据存在系统性低估，这是由其市场运作机制决定的，不代表实际市场活跃度。** 原因如下：

1. **短周期滚动市场**: Chainlink 事件均为加密资产（BTC/ETH/SOL/XRP）的短时价格预测盘口，时间窗口通常为 **5 分钟至 4 小时**（如 "Bitcoin Up or Down - February 25, 4:30AM-4:45AM ET"）
2. **快速轮换创建**: 每个时间窗口对应一个独立事件，到期后立即创建新的时间窗口事件；API 快照只能捕获到**当前活跃的时间窗口**
3. **交易量仅反映单窗口**: 采集到的交易量仅代表该特定时间窗口内的累计成交，而非该资产在 Chainlink 预言机下的历史总交易量
4. **大量零交易事件**: 由于新窗口持续创建，许多事件在被采集时尚未产生任何交易

与之对比，UMA 预言机下的事件（如 "Will Trump win the 2028 election?"）通常是长周期事件，交易量为**整个事件生命周期的累计值**，可能跨越数月甚至数年。

**因此，直接对比 UMA 与 Chainlink 的事件级交易量在统计学上不具有可比性。**

## 五、UMA 与 Chainlink 的本质差异

### 5.1 市场结构对比

| 维度 | UMA | Chainlink |
|------|-----|-----------|
| **市场类型** | 事件驱动型（离散结果预测） | 价格驱动型（连续价格走势） |
| **时间跨度** | 数天至数年 | 5 分钟至 4 小时 |
| **子市场结构** | 多结果盘口（平均 8.9 个子市场/事件） | 单一二元盘口（恒定 1 个子市场/事件） |
| **交易量性质** | 累计全生命周期交易量 | 仅当前时间窗口交易量 |
| **事件创建方式** | 人工/社区提案创建 | 系统自动滚动创建 |
| **裁决来源** | UMA 乐观 Oracle 人工裁决 | Chainlink 去中心化喂价自动裁决 |
| **覆盖分类** | 全分类（48 个） | 仅 Crypto（4 种资产） |

### 5.2 交易量不可直接对比的根因

```
UMA 事件生命周期:     |=============================>|   累计交易量 = $1,298,459 (平均)
                      创建                    结束
                      (数周~数月)

Chainlink 事件生命周期: |=>|                             单窗口交易量 = $614 (平均)
                        创建 结束
                        (5~30分钟)

问题: 爬虫在任意时刻快照只能捕获「当前活跃」的 Chainlink 窗口
     而 UMA 事件的交易量是整个生命周期的累积值
     → Chainlink 的真实日交易量 = Σ(所有已结束窗口交易量) ≫ 单窗口快照值
```

**因此，本报告表格中 Chainlink 的交易量数据 ($359,616) 仅反映采集瞬间的活跃窗口总和，而非 Chainlink 预言机在 Polymarket 上的真实历史贡献。如需准确对比，需按时间序列连续采集并累加每个窗口的最终交易量。**

## 七、结论与建议

### 7.1 核心结论

1. **Polymarket 市场集中度极高**: Politics 和 Sports 两个分类贡献了 86.7% 的总交易量，呈现典型的"二八分布"
2. **UMA 是绝对主导的预言机**: 覆盖 87% 的事件、98.3% 的子市场、99.99% 的交易量
3. **Chainlink 的市场定位差异化**: 专注于加密资产短时价格预测，采用自动化滚动创建模式，与 UMA 的人工事件驱动模式形成互补
4. **Chainlink 交易量数据的天然局限**: 由于短周期滚动窗口机制，单次 API 快照无法反映其真实交易规模。当前系统采集的 Chainlink 交易量 **被系统性低估了数个数量级**

### 7.2 数据改进建议

| 优先级 | 改进方向 | 预期效果 |
|:-----:|---------|---------|
| P0 | 增加 Chainlink 事件的连续时序采集，累加每个窗口最终交易量 | 获取 Chainlink 真实日/周/月交易量 |
| P1 | 接入 KaIshi API，扩展跨平台对比维度 | 完成完整的 Polymarket vs KaIshi 对比分析 |
| P2 | 增加历史事件（已关闭事件）的采集 | 获取 Polymarket 全生命周期数据视角 |
| P3 | 按分类维度交叉分析预言机使用偏好 | 理解不同场景下预言机选型逻辑 |
